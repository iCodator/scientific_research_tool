BOOLEAN QUERY:
  A search with AND, OR, NOT operators
  Example: (cancer OR tumor) AND therapy

UNIVERSAL FORMAT:
  Query format that works for any database
  System translates to database-specific syntax automatically

FIELD TAG:
  Brackets restricting search to specific field
  Example: [TIAB] = Title/Abstract, [MeSH] = Medical Subject Heading

SINGLETON PATTERN:
  Design rule: Only one instance of something exists
  Example: Only one LoggingManager per database

ADAPTER PATTERN:
  Design rule: Create wrapper for different implementations
  Example: PubMedAdapter, EuropePMCAdapter, CochraneAdapter all implement
           DatabaseAdapter interface

QUERY COMPILATION:
  Process of converting universal query to database-specific syntax
  Example: (year1:year2[pdat]) â†’ PUB_YEAR:(year1-year2) for Europe PMC

A: Usually settings.py to configure API credentials

Q: How do I add a new database?
A: Create new adapter inheriting from database_adapter.py, implement search()

Q: Why are there 2 JSON files?
A: PubMed and Europe PMC use different syntax - each needs own rules

Q: What if a query returns 0 results?
A: Check IMPORTANT WARNINGS section - most likely syntax error

Q: How are logs organized?
A: Separate log file per database, per day. See logging_manager.py

Q: How is query validation done?
